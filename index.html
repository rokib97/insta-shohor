How does JavaScript work?

When we write code, to complie the code to machine language, browser takes the help of JavaScript Engine like V8(Chrome Engine)which complied our code to machine language then we can see our output.

Inside the V8 engine with the help of JIT compiler ,When JavaScript code runs for the first time it creates a Global Execution Context(Window Object) where also have two phases, First is loading or memory creation phase where Js set the memory for variables(values set to undefined which is called hoisting) and functions(reference is created).Second phase is execution where Js again start the code from the first line and set all the value of varibale which was undefined in the first phase, When there is a function called or invocked in the Global execution a new Function execution context is created just like the Global and it also have the same two phase memory creation and execution. when the function execution executed its inside code then it will vanish and js will back to global and execute the rest code. And Finally Global execution will finish when there is nothing to execute now.

JavaScript follows Stack data structure(LIFO) to procedure these execution context which called Call Stack. where the main or Global execution is the First in then function execution context is placed, when a function execution is done it popped out from the stack. and in the end when there are no function execution ramain the Global also popped out from there and the program ends.

What is Event loop in JavaScript?

Inside Browser There is a JavaScript Engine(V8 for chrome) and an environment to run JavaScript properly. This Engine has two parts, Heap And Call Stack. Engine has Some Assistant called Web API's and Callback Queue.

JavaScript is single threaded language but it executes asynchronously multiple task with the help of event loop and callback queue.

The Event Loop always monitor the Call Stack and the Callback Queue.We know synchronous task executes through the call stack but when a asynchronous task came out call Stack takes that and pass it to the Callback Queue with the help of Web APIs. and executes the rest of the code. When the synchronous task is executed then The Event loop check the queue and call stack if call stack is free and queue has some task it just takes the task in a FIFO way and pass it to the callback. Callback then executes these asynchronous task. If there is any Promise in the code , Event loop takes that promise from Microtask Queue even before callback queue and pass to the call stack. And Thats how Js executes asynchronous code using Evnet Loop.

What are the differences between Session storage and Local storage?

Local Storage:

  • Capacity of local Storage is 10 mb

  • Accessing Local Storage from any window is possible.

  • Local Storage never expires unless the user delete it manually

Session Storage:

  • Capacity of Session Storage is 5 mb

  • Accessing Session Storage possible on within the Tab, not from any windows

  • Session Storage expires On tab close
